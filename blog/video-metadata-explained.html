動画のメタデータとは — ファイルに埋め込まれた見えない情報

セキュリティ プライバシー

動画処理のツールを作ってると、動画ファイルの中に映像と音声以外のデータがいろいろ入ってることに気づく。撮影日時、カメラの機種名、GPS座標——再生画面には一切出てこないけど、ファイルの中にはしっかり残ってる。これがメタデータ。自分がffmpegで動画を扱ってるときに「こんなの入ってるのか」と驚いたのがきっかけで、ちょっと調べてみたらプライバシー的にかなり重要な話だった。

メタデータって何が入ってるのか

動画ファイルに埋め込まれるメタデータは、思ってる以上に多い。スマホで撮った動画をffmpegで覗くと、だいたいこんな情報が出てくる。

カテゴリ含まれる情報リスク
位置情報GPS座標（緯度・経度）、高度撮影場所＝自宅がバレる可能性
日時作成日時、更新日時行動パターンの推定
デバイス情報カメラの機種名、メーカー、OS端末の特定
撮影設定解像度、フレームレート、ビットレートリスク低（技術情報）
ソフトウェアエンコーダ名、編集ソフト名リスク低

写真の世界だとEXIFという規格が有名で、GPSタグ問題は結構前から知られてた。でも動画にも似たような情報が入ってることは、あまり意識されてない気がする。

実際にメタデータを見てみる

ffmpegがインストールされてるなら、ターミナルで一発。

ffprobe -v quiet -print_format json -show_format -show_streams video.mp4

これを実行すると、JSONでずらっとメタデータが出てくる。自分が最初にこれを試したときは、iPhoneで撮った動画から com.apple.quicktime.location.ISO6709 というキーでGPS座標がそのまま出てきて、Googleマップに貼ったら自宅の位置がピンポイントで表示されて少しゾッとした。

もっと手軽に見たいなら：

ffprobe -v quiet -show_entries format_tags video.mp4

これだとタグ情報だけが出るので見やすい。creation_time（作成日時）、encoder（エンコーダ）、handler_name（ハンドラ名）あたりが表示される。

GPS情報が残るケース

スマホのカメラアプリは、設定で位置情報をオンにしてると撮影時のGPS座標を動画に埋め込む。iPhoneだと「設定 → プライバシー → 位置情報サービス → カメラ」で制御できるけど、デフォルトでオンになってることが多い。

問題は、このGPS情報が入ったまま動画をどこかにアップロードした場合。アップロード先がメタデータを消してくれるかどうかはサービス次第。

SNS別のメタデータ除去状況

サービスGPS除去EXIF除去備考
Twitter/X除去される除去される再エンコードされるので元データはほぼ残らない
YouTube除去される除去されるただし管理画面で撮影場所を設定すると公開される
Instagram除去される除去される写真・動画ともにメタデータはストリップされる
LINE除去される除去される送信時に再圧縮。「オリジナル画質」送信時は要注意
メール添付そのまま残るそのまま残る一切加工されない。一番危険
クラウドストレージそのまま残るそのまま残る共有リンク経由だとメタデータ付きでダウンロードされる

大手SNSは基本的にメタデータを削除してくれる。ただ、メールに添付して送ったりGoogleドライブで共有したりすると、メタデータは丸ごと残る。仕事で動画ファイルをメールでやり取りするとき、撮影場所の情報が入りっぱなしだったりするので地味に危ない。

メタデータを削除する方法

ffmpegで削除

自分が一番よく使うのはffmpeg。コマンド一発でメタデータを全部消せる。

ffmpeg -i input.mp4 -map_metadata -1 -c copy output.mp4

-map_metadata -1 が「メタデータを一切コピーしない」の意味。-c copy は映像と音声を再エンコードせずコピーするので、画質の劣化なし・処理も一瞬で終わる。

GPS情報だけ消したい場合はちょっとややこしくて、QuickTimeのlocation tagだけを消す方法もあるんだけど、正直全部消した方が安全だし楽。

ExifToolで削除

ExifToolは写真向けのツールとして有名だけど、動画のメタデータも扱える。

exiftool -all= video.mp4

これで全メタデータが消える。写真と動画を一括処理したいときに便利。ただ、動画のコンテナ内部の一部メタデータ（ストリーム情報とか）はExifToolだと消しきれないこともあるので、完全にクリーンにしたいならffmpegの方が確実。

スマホの設定で撮影時から対策する

そもそも撮影時に位置情報を埋め込まないのが一番簡単。

  • iPhone: 設定 → プライバシーとセキュリティ → 位置情報サービス → カメラ → 「なし」
  • Android: カメラアプリの設定 → 「位置情報タグ」をオフ（メーカーによって場所が違う）

ただし、位置情報をオフにすると写真アプリの「撮影地マップ」みたいな機能が使えなくなる。個人的には位置情報はオンのまま撮影して、外部に共有するときだけメタデータを消す運用にしてる。

メタデータで実際に起きた問題

「そんなの気にしすぎでは」と思うかもしれないけど、実際にメタデータが原因で問題になったケースはいくつかある。

  • 2012年、McAfeeの創業者ジョン・マカフィーの所在地バレ: 中米で逃亡中だったマカフィー氏を取材したジャーナリストが、iPhoneで撮った写真をそのままブログに載せた。EXIF内のGPS情報からグアテマラの具体的な住所が特定されて、翌日に逮捕
  • 写真・動画のGPS情報からストーカー被害: SNSにアップした画像のメタデータから自宅を特定されるケースは、日本でも複数報告されてる。今は主要SNSがメタデータを消してくれるから減ったけど、個人ブログやファイル共有サービス経由だと今でもリスクがある

このサイトでのメタデータの扱い

うちのサイトでTwitter動画をダウンロードする場合、取得されるのはTwitterが再エンコードした後の動画。元の投稿者がスマホで撮った動画にGPS情報が入っていたとしても、Twitterのサーバー側で再エンコードされる過程でメタデータは除去されてる。なので、ダウンロードしたMP4に位置情報が残ってることは基本的にない。

ただ、エンコーダ情報（Lavf とか）や作成日時みたいな技術的なメタデータは残ってることがある。気になる人は、上に書いたffmpegのコマンドで消してから使うのがいいと思う。

まとめ

動画ファイルには映像と音声以外にもいろんな情報が埋まってる。特にGPS情報は撮影場所がピンポイントで分かるので、扱いに注意が必要。大手SNSは自動で除去してくれるけど、メール添付やクラウド共有ではそのまま残る。

自分はffmpegで動画を処理するようになって初めてメタデータの存在をちゃんと意識するようになった。ffprobe で一度自分の動画のメタデータを覗いてみると、何が入ってるか分かって良いと思う。知った上で対処するのが一番確実。
