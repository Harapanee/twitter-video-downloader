MP4だけじゃない — 知っておきたい動画ファイル形式まとめ

技術解説 動画形式

ダウンロードツールを作るとき、対応する出力形式をどうするかで結構悩んだ。MP4だけでいいのか、WebMにも対応すべきか、MKVは？AVIは？そのあたりを判断するために各形式の特性をかなり調べまくったので、その時の知見をまとめておく。

先に結論を書いておくと、一般的な用途ならMP4を選んでおけばまず間違いない。ただ、他の形式にもそれぞれ存在意義があって、知っておくと「なんでこの形式なんだ？」という場面で困らなくなる。

そもそも「動画ファイル形式」とは

最初にここをはっきりさせておきたいんだけど、よく「MP4形式の動画」と言うときの「MP4」はコンテナフォーマットのことであって、映像の圧縮方式（コーデック）のことではない。

コンテナは名前の通り「入れ物」で、映像データ、音声データ、字幕データ、メタデータ（タイトルとか著者とか）を1つのファイルにまとめて格納する役割を持つ。同じMP4コンテナでも、中の映像がH.264で圧縮されてるものもあればH.265のものもある。

ffmpegを使い始めた頃、この「コンテナ」と「コーデック」の区別が曖昧で何度かハマった。-c copy（中身のコーデックはそのまま、コンテナだけ変える）と -c:v libx264（映像を再エンコードする）は全然別の処理なのに、どっちも「変換」と呼ばれてて最初は混乱した。

主要な動画ファイル形式

MP4（.mp4）

言わずと知れた最強の汎用フォーマット。正式名称はMPEG-4 Part 14。

何がすごいかって、対応デバイスの幅が異常に広い。Windows、Mac、iPhone、Android、ゲーム機、カーナビ、大体何でも再生できる。Web上でもHTML5の <video> タグでネイティブ再生に対応してる。映像コーデックはH.264が一般的で、最近はH.265やAV1も格納できる。

うちのダウンロードツールでMP4を出力形式にしてるのは、単純にユーザーが一番困らないから。「ダウンロードしたけど再生できません」という問い合わせを限りなくゼロにしたい。MP4ならまずそういうことは起きない。

WebM（.webm）

Googleが推進してるオープンフォーマット。映像コーデックはVP8、VP9、またはAV1。音声はVorbisかOpus。

Web向けに最適化されていて、ChromeとFirefoxではネイティブ対応。ファイルサイズがMP4より小さくなるケースもあって、Web上での配信には結構優秀。YouTubeの内部でもWebM（VP9）が使われてる。

ただしApple系との相性が悪い。Safariでの対応が遅れてて、iPhoneで再生できないケースがある。これが痛い。うちのツールでWebM出力を見送ったのはまさにこれが理由で、iPhoneユーザーから「再生できない」と言われるのが目に見えてた。

MKV（.mkv）

Matroska Video。名前の由来はロシアのマトリョーシカ人形。「何でも入る」コンテナとして設計されていて、H.264、H.265、VP9、AV1、字幕トラック、複数音声トラック、チャプター情報、ほぼ何でも格納できる。

アニメや映画をコレクションしてる人はMKV推しが多い印象。字幕を複数言語入れたり、音声を吹き替えと原語で切り替えたりできるのが便利。VLCやMPC-HCみたいなデスクトップの動画プレーヤーとの相性が抜群にいい。

ただしスマホやブラウザでの対応はイマイチ。HTML5の <video> タグはMKVに対応してないし、iPhoneのデフォルトプレーヤーでも再生不可。ツール開発の観点からすると、出力形式としては採用しにくい。

AVI（.avi）

Microsoftが1992年に開発した歴史あるフォーマット。正式名称はAudio Video Interleave。

正直、2026年にAVIを積極的に使う理由はほぼない。ストリーミングに対応してない、字幕トラックの扱いが貧弱、可変ビットレートとの相性が悪い。ファイルサイズも他のフォーマットに比べて大きくなりがち。

ただ古いデバイスや業務用の機器でAVIしか受け付けないケースが稀にある。あと、録画ソフトのデフォルト出力がAVIになってることもまだある。見かけたらMP4に変換しておけば間違いない。

MOV（.mov）

AppleのQuickTimeフォーマット。iPhoneで撮影した動画はデフォルトでMOV。中身はMP4とほぼ同じ（どちらもISO Base Media File Formatがベース）なんだけど、Apple独自の拡張が入ってたりする。

Mac/iPhoneユーザーは意識してなくてもMOVを大量に扱ってるはず。Windowsでも再生自体はできるけど、たまに互換性の問題が出る。特にHEVC（H.265）で撮影したMOVファイルを古いWindows機で開こうとすると再生できないことがある。

ffmpegだと -c copy でMP4に変換するのが一瞬で終わる。コーデックは同じだからコンテナの差し替えだけ。実質的にはMP4とほぼ同じものだと思っていい。

TS（.ts）

Transport Stream。HLS配信で使われるセグメントファイルの形式。うちのツールを使ってて一番馴染みがあるのが実はこれ。

TSは放送用に設計されたフォーマットで、エラー耐性が高い。電波が乱れても途中から復帰できるように作られてる。テレビ録画した .ts ファイルを持ってる人もいると思うけど、あれがこれ。

HLSでは動画を数秒ごとのTSセグメントに分割して配信する。うちのツールではこのTSセグメントを全部集めて結合し、ffmpeg.wasmでMP4コンテナに入れ直してる。MP4変換に失敗した場合はTSのまま保存するフォールバックも用意してある。VLCなら.tsファイルもそのまま再生できる。

結局どれを使えばいいのか

用途別にまとめると：

  • SNSにアップする・人に送る → MP4。考える余地なし
  • Webサイトに埋め込む → MP4。WebMも併用するとChromeで軽くなる場合がある
  • 動画コレクション（映画・アニメ） → MKV。字幕と複数音声を入れられる
  • iPhone撮影→Mac編集 → MOV。Apple同士ならそのまま使える
  • テレビ録画 → TS。録画ソフトが出力するのでそのまま

動画ダウンロードの文脈で言えば、保存したファイルが.tsになってる場合は、MP4に変換しておくと後で困らない。うちのツールではHLS動画を自動でMP4変換してるけど、変換に失敗した場合はTSで保存するので、その場合はVLCで再生するか、ffmpegでMP4に変換してほしい。
